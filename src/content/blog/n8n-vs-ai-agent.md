**直接回答：如果你是零基礎新手，做 Threads / IG / FB 自動發文請先用 AI Agent（Claude Code 或 Codex），而不是 n8n。** 理由很簡單：n8n 要你自己懂每個節點在做什麼；AI Agent 是你用中文描述需求，它把技術細節做完。

## 兩條路線的差別

| | n8n | AI Agent（Claude Code / Codex） |
| --- | --- | --- |
| 上手門檻 | 要理解節點、憑證、Webhook | 會打中文就能開始 |
| API 申請 | 自己看文件申請、填憑證 | AI 帶著你申請、幫你填設定 |
| 出錯時 | 自己排查節點 | 把錯誤訊息丟給 AI 處理 |
| 彈性 | 節點有什麼功能就做什麼 | 要什麼功能就叫 AI 寫什麼 |
| 費用 | 自架免費／雲端方案月費 | AI 訂閱每月 20 美元起 |

## 為什麼新手常在 n8n 卡關？

搜尋「n8n threads 自動發文」的人很多，但實際跑通的新手很少，卡點幾乎都一樣：

1. **Meta API 申請與帳號綁定**——這關跟 n8n 無關，n8n 也不會幫你（[申請教學看這篇](/blog/meta-api-application/)）
2. **憑證與 token 設定**——填錯一個欄位整條流程都是紅的，而 n8n 的錯誤訊息對新手並不友善
3. **流程邏輯**——你得自己知道「先取 token → 建立媒體容器 → 發布」這種順序，n8n 只給你積木，不告訴你怎麼拼

AI Agent 的差別在於：這三關它全程陪你走，你[用中文下指令、它執行](/blog/no-tech-background/)，錯了直接把訊息丟回去給它修。

## 什麼人適合直接用 n8n？

- 已經有工程背景、看得懂 API 文件的人
- 公司已有 n8n 環境、你只是要加一條流程
- 流程極度固定、不需要 AI 產內容的純搬運任務

如果你不在上面三種裡，先用 AI Agent 把[自動發文流程](/blog/auto-posting-permissions/)跑通一次，之後要不要搬去 n8n 都是你的自由——那時候你已經看得懂整條流程了。
